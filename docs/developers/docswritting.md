# 文档编写指南

> [!TIP]
>
> 该项目目前没有文档站，所有的文档编写均面向开发者且采用 Markdown 格式，你可以前往 [Markdown 说明](https://markdown.com.cn/) 查看更详细的关于 Markdown 格式的说明

## 文档结构约定

- **开发向文档**：放在 `docs/developers/`，面向贡献者，侧重开发流程、规范、接口说明。
- 单篇文档建议控制篇幅，过长时可拆成多篇并用目录/链接串联。

## 写作风格

- **简洁明了**：先结论后细节，避免冗长铺垫。
- **统一用语**：同一概念全文使用同一术语（如「运行日志」不要与「执行日志」混用）。

## 常用格式

### 标题层级

- 一级标题（`#`）仅用于文档标题，一篇文档一个。
- 二级标题（`##`）划分大节，三级（`###`）及以下用于小节与子项。
- 标题层级不要跳级（例如不要从 `##` 直接到 `####`）。

### 提示块

可选用以下块语法，保持与仓库内其他文档一致：

- `> [!TIP]`：提示、小技巧。
- `> [!IMPORTANT]`：重要、必读。
- `> [!WARNING]`：注意、风险。
- `> [!NOTE]`：补充说明。

### 代码与路径

- 行内代码使用反引号：`debug`、`vmlinux`。
- 多行代码或命令使用围栏代码块，并标注语言（如 \`\`\`bash\`\`\`、\`\`\`json\`\`\`）。
- Windows 路径中的反斜杠在 Markdown 中可写为 `\\` 或使用正斜杠，避免被转义。

### 列表与步骤

- 无序列表用 `-`，有序步骤用 `1. 2. 3.`。
- 操作类内容建议用有序列表，方便读者按步执行。
